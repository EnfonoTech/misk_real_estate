End-to-end process from first contact to payment collection.

↓

Sales Order is not used. The Quotation replaces Sales Order entirely. After approval (Confirmed status), the sales team creates Property Bookings directly from the Quotation.
Quotation & Approval Workflow
Sales team creates a Quotation per deal. It goes through 2-stage approval before a Property Booking can be created.

Approval States

Draft
→ Submit for Approval Misk Sales
↓
Pending Sales Approval
→ Approve / Reject Sales Manager
↓
Pending Finance Approval
→ Approve / Reject Misk Finance
↓
Confirmed
→ Ready for Property Booking creation
If Rejected at any stage, the Quotation returns to Draft. The sales team can edit and resubmit.

2

Submit for Approval

Save the Quotation, then click Submit for Approval. Status changes to Pending Sales Approval. The Quotation is locked for editing until a decision is made.

3

Two-Stage Approval

Sales Manager approves → Finance Manager approves → status becomes Confirmed. Either approver can reject at their stage.

4

Create Property Booking (after Confirmed)

A button appears: Create Property Booking.

  • The system creates one Property Booking per line item
  • If the Quotation is linked to a Lead, the Lead is automatically converted to a Customer — no manual step needed
  • Each booking is submitted immediately with the PDC schedule generated
  • If any unit is already Booked or Sold, that line is skipped with a warning

2

Save and Submit

Click Save to review, then Submit to confirm the booking.

On submit, the system automatically generates the PDC Schedule — a table of all future cheques with dates, amounts, and a PDC checkbox.

For a 12M plan starting June 2026:

  • Row 1 — Booking cheque (today)
  • Row 2 — Down payment cheque (booking date + Days from Settings)
  • Rows 3–14 — 12 monthly installment cheques
3

PDC Schedule — PDC Checkbox

Each row has a PDC checkbox (default: checked).

ColumnDescription
PDC (checkbox)Checked = a PDC Entry will be created for this row. Unchecked = skip — use for rows where the customer pays directly without a cheque.
Cheque DateDate the cheque is due
AmountCheque amount
Cheque NoCheque number. Mandatory on submit when PDC is checked. Not required on save — fill in before submitting.
StatusMirrors the linked PDC Entry status
Cheque No validation runs only on submit. You can save an incomplete schedule and add numbers later.
4

Create PDC Entries

After submit, click PDC → Create PDC Entries.

  • Creates one PDC Entry per schedule row where PDC checkbox is checked
  • Rows with PDC unchecked are skipped entirely
  • Each PDC Entry starts as Pending
This button appears only once. After PDC Entries are created it is hidden. Use View → PDC Entries to see them.
5

Other Buttons After Submit

View dropdown

  • PDC Entries — PDC Entry list filtered to this booking
  • Sales Invoices — Sales Invoice list filtered to this booking
  • Payment Entries — Payment Entry list filtered to this booking

Actions dropdown

  • Generate Invoices Now — only for All at Once bookings where invoices haven't been generated; re-queues the background job
  • Mark Unit Sold — only when all PDC rows are Cleared or Cancelled; marks unit as Sold and booking as Closed

Invoice Generation Mode

Controls when Sales Invoices are created. Locked after submit. Default: Monthly (Cron).

OptionBehaviour
Monthly (Cron)
default		 A scheduled job runs on the 5th of each month. Creates one Sales Invoice per PDC row due that month. Invoices appear automatically — no manual creation needed.
All at Once On booking submit, a background job creates all invoices immediately — one per PDC schedule row. If the job fails, use Actions → Generate Invoices Now to retry.
A Sales Invoice must be linked to a PDC Entry before it can be marked Cleared. If no invoice exists yet, generate it first (Actions → Generate Invoices Now, or wait for the monthly cron).

  • Mark Cleared: Requires linked Sales Invoice. Creates a Payment Entry and posts GL. The bank account is resolved from the PDC Entry's Mode of Payment → Mode of Payment Account → default_account (falls back to company's default bank account).
  • Mark Bounced: No GL entries. Invoice stays outstanding. Follow up with customer for a replacement cheque.
  • Record Manual Payment: Customer pays by cash/transfer instead of cheque. Requires linked Sales Invoice. Creates Payment Entry, cancels PDC Entry, marks PDC schedule row as Cleared.
  • Post GL Entry: Only when Cleared but GL not posted (status set manually). Creates the missing Payment Entry.
Never manually change the Status dropdown to "Cleared". Always use Actions → Mark Cleared. Manual change skips Payment Entry creation and leaves GL unposted. Use Post GL Entry to recover if this happened.

2

Submit the Batch

Click Submit to lock the batch.

All PDC Entries in the batch automatically move from Pending → In Batch.
3

Mark Sent to Bank

After physically delivering the cheques: open the batch → Actions → Mark Sent to Bank → Confirm.

All cheques move to Deposited status.
4

Clear Individual Cheques

After bank confirms each cheque:

  • Click View → PDC Entries on the batch
  • Open each Deposited PDC Entry
  • Verify a Sales Invoice is linked, then click Actions → Mark Cleared
PDC Entry — Manual Payment
When a customer cancels a PDC cheque and pays instead by cash or bank transfer.

When to Use This

Use when a customer cannot honour a specific cheque and will pay the same amount by cash or bank transfer. This cancels the cheque record and immediately settles the linked Sales Invoice.
1

Prerequisites

  • PDC Entry status is Pending, Deposited, or In Batch
  • A Sales Invoice is linked to the PDC Entry
If no Sales Invoice is linked, generate invoices first via the booking (Actions → Generate Invoices Now).
2

Click Actions → Record Manual Payment

FieldDescription
Mode of PaymentCash, Bank Transfer, or any configured mode
Payment DateDate received
AmountDefaults to PDC Entry amount
NotesOptional reason / reference number

Click Create Payment Entry & Cancel PDC to confirm.

3

What Happens After Confirmation

  • A Payment Entry is created against the linked Sales Invoice
  • PDC Entry status → Cancelled
  • PDC Schedule row on the booking → Cleared

Unit Import (Excel)
Bulk-import units (Items) from an Excel spreadsheet. Used for initial setup of a building's unit list.

Download the Template First

Always use the provided template — it has the correct column layout and the building name input cell pre-formatted.

Template Structure

LocationContent
Cell B1Label: "Building Name:" — do not edit this cell
Cell C1Input (light green) — type the building name here (e.g. Al Misk Tower). This becomes the Item Group for all units in the sheet.
Row 3Column headers row
Row 4 onwardsOne unit per row
Cell C1 (building name) is required. If blank, the import will show an error before uploading.

Import Steps

  1. Download the template Excel file from the import tool
  2. Type the building name in cell C1
  3. Fill unit data from row 4: Unit Code, Floor, Unit Type, Area (Sq Ft), Standard Rate
  4. Save the file and upload via the import tool
  5. System reads C1 for the Item Group and creates all units automatically
