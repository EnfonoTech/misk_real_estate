2

Save and Submit

Click Save to review, then click Submit to confirm the booking.

On submit, the system automatically generates the PDC Schedule — a table of all future cheques with dates and amounts.

For a 12M plan starting May 2026:

  • Row 1 — Booking cheque (today)
  • Row 2 — Down payment cheque
  • Rows 3–14 — 12 monthly installment cheques
3

Create PDC Entries

After submit, a button appears: PDC → Create PDC Entries

  • Click the button
  • Confirm the dialog (shows count of schedule rows)
  • System creates one PDC Entry record per schedule row
  • Each PDC Entry has: cheque number, customer, amount, due date, status = Pending
This button appears only once. Once PDC Entries are created, the button is hidden. Use View → PDC Entries to see them.
4

Buttons After Submit

After submission, the following dropdowns appear on the booking form:

View dropdown — 3 items

  • PDC Entries — opens the PDC Entry list filtered to this booking
  • Sales Invoices — opens the Sales Invoice list filtered to this booking
  • Payment Entries — opens the Payment Entry list filtered to this booking

Actions dropdown

  • Generate Invoices Now — only visible for All at Once bookings where Sales Invoices have not yet been created; re-queues invoice generation as a background job
  • Mark Unit Sold — only visible when all PDC schedule rows are Cleared or Cancelled; marks the unit as Sold and the booking as Converted

PDC group

  • Create PDC Entries — only visible when no PDC Entries have been created yet; creates one PDC Entry record per schedule row

Invoice Generation

Select field (required) that controls how and when Sales Invoices are created for this booking's PDC schedule rows. The field is locked after the booking is submitted and cannot be changed. Default: Monthly (Cron).

OptionBehaviour
Monthly (Cron)
default — best for ongoing tracking		 A scheduled job runs on the 1st of each month. For every PDC schedule row where cheque_date falls in the current month and no Sales Invoice exists yet, the system auto-creates one Sales Invoice. Finance team reconciles as invoices appear — no manual creation needed. Best for bookings where you want invoices to match each payment period.
All at Once
full invoice register from day one		 On booking submit, a background job creates all Sales Invoices immediately — one per PDC schedule row. Each invoice gets posting_date = booking date and due_date = that row's cheque_date. If the background job fails (e.g. timeout), use the Actions → Generate Invoices Now button on the booking to re-queue generation.
Invoice Generation cannot be changed after submit. Choose the correct mode before submitting the booking.

Mark Unit Sold

Finalises the property transaction once every cheque in the PDC schedule has been cleared by the bank.

1

Prerequisite — All PDC Rows Cleared

The button is visible in Actions only when every row in the PDC Schedule table has status Cleared or Cancelled. If any row is still Pending, Deposited, Bounced, or In Batch, the button remains hidden.

2

Click Actions → Mark Unit Sold

Opens a confirmation dialog. Confirm to proceed.

Two things happen simultaneously: the Unit's status is set to Sold in the Item master, and the Booking status moves to Converted.
3

After Mark Unit Sold

  • Unit is no longer bookable — system blocks any new booking against it
  • Unit appears as Sold in the Unit Management list and the unit availability filter
  • Booking record is read-only; no further PDC or invoice actions are needed

Unit Status — Values & Transitions

StatusMeaningHow it is set
Available Unit is free to be booked Default when unit is created, or automatically when a booking is cancelled
Booked Unit has an active booking Automatically on Booking Submit
Sold All cheques cleared; ownership transferred Automatically via Actions → Mark Unit Sold (available only when all PDC schedule rows are Cleared or Cancelled)
Reserved Unit is held but not formally booked Set manually by Admin on the Item record
Under Maintenance Unit is temporarily unavailable Set manually by Admin on the Item record
Double-booking protection: the system blocks submission of a Property Booking if the selected unit's status is Booked or Sold. An error message is shown and the booking cannot be submitted until a different unit is selected.

Actions Available by Status

The action buttons visible on a PDC Entry depend strictly on its current Status value. Refer to this table before taking any action.

Deposited ✅ ✅ ✅ (only if SI linked) ❌

Cleared (GL NOT posted) ❌ ❌ ❌ ✅

Cancelled ❌ ❌ ❌ ❌
  • Mark Cleared: Creates a Payment Entry (GL posts here). Sets GL Posted = ✓. Can only run when status is In Batch or Deposited.
  • Mark Bounced: Marks cheque as returned. Invoice stays outstanding. User must follow up with the customer for a replacement cheque or alternative payment.
  • Record Manual Payment: Use when customer cancels their cheque and pays by cash or bank transfer instead. Requires a Sales Invoice to be linked to this PDC Entry. Creates a Payment Entry, cancels the PDC entry, and marks the PDC schedule row on the booking as Cleared.
  • Post GL Entry: Appears only when status is Cleared but GL Posted is unchecked — this happens when status was set manually instead of using Actions → Mark Cleared. Use this button to create the missing Payment Entry and post GL.
⚠️ Never manually change the Status dropdown to "Cleared". Always use Actions → Mark Cleared. Manual status change skips Payment Entry creation and leaves GL unposted — visible as an unchecked GL Posted checkbox on the record. Use Actions → Post GL Entry to recover if this has already happened.
1

Mark Cleared (on clearance day)

Used by Finance team when the bank confirms a cheque has cleared.

  1. Open the PDC Entry record
  2. Click Actions → Mark Cleared
  3. In the dialog: enter the Clearance Date
  4. Click Mark Cleared to confirm
System automatically creates a Payment Entry and posts GL entries. This is the only point where accounting entries are created (B7 compliance).

Available when status is Deposited or In Batch.

2

Mark Bounced (returned cheque)

Used when the bank returns a cheque due to insufficient funds or account issues.

  1. Open the PDC Entry record
  2. Click Actions → Mark Bounced
  3. In the dialog: enter reason/notes (optional)
  4. Click Mark Bounced to confirm
No GL entries are created. The invoice remains outstanding. Follow up with customer for replacement cheque.
3

View Links on PDC Entry

Navigation buttons available on every PDC Entry form:

  • View → Property Booking — go to the parent booking
  • View → PDC Batch — go to the linked batch (if batched)
  • View → Sales Invoice — linked invoice (when generated)
  • View → Payment Entry — linked payment (after clearing)

Create a New Batch

Go to PDC Management → PDC Batch → New

  • Set Bank Account — the target bank account

3

Mark Sent to Bank

After physically delivering cheques to the bank:

  1. Open the submitted PDC Batch
  2. Click Actions → Mark Sent to Bank
  3. Confirm the dialog
All cheques in the batch move to Deposited status. Finance team can then mark each as Cleared when bank confirms.
4

Clear Individual Cheques

After bank sends clearance confirmation for each cheque:

  • Click View → PDC Entries on the batch
  • Open each Deposited PDC Entry
  • Click Actions → Mark Cleared (see PDC Entry section above)
PDC Entry — Manual Payment
When a customer cancels a PDC cheque and pays instead by cash or bank transfer.

When to Use This

Use this when a customer calls to say they cannot honour a specific cheque and will pay the same amount by cash or bank transfer instead. This cancels the cheque record and immediately settles the linked Sales Invoice.
1

Prerequisites

The Record Manual Payment action is visible only when:

  • PDC Entry status is Pending, Deposited, or In Batch
  • A Sales Invoice is linked to the PDC Entry
If the PDC Entry has no linked Sales Invoice yet, generate invoices first via the booking (View → Sales Invoices, or Actions → Generate Invoices Now).
2

Open the PDC Entry

From the Property Booking form: click View → PDC Entries, then open the specific PDC Entry for the cheque to be replaced.

3

Click Actions → Record Manual Payment

A dialog opens. Fill in:

FieldDescription
Mode of PaymentCash, Bank Transfer, or any other mode configured in ERPNext
Payment DateDate the cash/transfer was received
AmountAmount received (defaults to the PDC Entry amount)
NotesOptional — reason for PDC cancellation or reference number

Click Create Payment Entry & Cancel PDC to confirm.

4

What Happens After Confirmation

All four updates happen atomically:
  • A Payment Entry is created against the linked Sales Invoice — the SI outstanding balance goes to zero
  • The PDC Entry status changes to Cancelled
  • The corresponding PDC Schedule row on the booking changes to Cleared with the Payment Entry reference
  • The Payment Entry is linked via the property_booking field for full traceability
PDC Reports
Three built-in reports for monitoring cheque status, aging, and monthly cash flow.

How to Run a Report

Open the Misk Real Estate workspace → click the report name → set filters in the top bar → press Run (or Enter).

PDC Status Report

Full list of all PDC Entries. Shows cheque details, status, booking, batch, and payment entry links. Includes a pie chart of amounts by status.

Filters
Customer Building Status From Cheque Date To Cheque Date

PDC Aging Report

Shows outstanding (non-cleared) cheques grouped by how overdue they are. Bar chart by aging bucket. Used for collections follow-up.

Aging Buckets
Not Yet Due 1–30 Days 31–60 Days 61–90 Days 90+ Days
Filters
As of Date ✱ Customer Building Status

PDC Monthly Forecast

Monthly cash flow forecast from PDC cheques. Stacked bar chart showing Pending, Deposited, Cleared, and Bounced amounts per month.

Filters
From Date ✱ To Date ✱ Building Customer

✱ Required filter

All reports support Export to Excel using the download button at the top right of the report page.
